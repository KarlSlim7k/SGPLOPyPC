Sistema para la Gestión del Procedimiento de Licitación de Obra Pública y Procesos de Contratación
1) Objetivo del sistema
Desarrollar un sistema web integral que permita gestionar, controlar y dar seguimiento a los procedimientos de licitación de obra pública y procesos de contratación, facilitando la administración de convocatorias, recepción de propuestas, evaluación de participantes y adjudicación de contratos.
El sistema tendrá como finalidad digitalizar y organizar el proceso administrativo, permitiendo mejorar la transparencia, trazabilidad y eficiencia en cada una de las etapas del procedimiento de contratación pública.
Asimismo, el sistema proporcionará un repositorio centralizado de información, donde se pueda consultar el historial de licitaciones, participantes, documentos técnicos y decisiones administrativas relacionadas con cada proceso.
2) Tipos de usuarios y permisos
El sistema deberá contemplar diferentes tipos de usuarios con permisos específicos de acuerdo con su rol dentro del proceso de contratación.
Usuario público / visitante
Podrá:
Consultar convocatorias de licitación publicadas.
Revisar información general de proyectos de obra pública.
Descargar bases de licitación y documentos informativos.
Consultar resultados de procesos de adjudicación.
Usuario proveedor / contratista registrado
Podrá:
Registrarse como proveedor o empresa contratista.
Consultar licitaciones disponibles.
Descargar bases y requisitos de participación.
Subir documentación requerida para participar en una licitación.
Enviar propuestas técnicas y económicas.
Consultar el estado de sus propuestas.
Recibir notificaciones sobre resultados o aclaraciones.
Usuario administrador / responsable de contratación
Podrá:
Registrar nuevas licitaciones o concursos.
Publicar convocatorias.
Gestionar documentación del proceso.
Evaluar propuestas recibidas.
Registrar resultados de evaluación.
Realizar la adjudicación del contrato.
Gestionar proveedores y usuarios.
Generar reportes y estadísticas.
Supervisar la trazabilidad de todo el proceso.
3) Módulos principales del sistema
El sistema deberá estructurarse en varios módulos funcionales que permitan administrar cada etapa del procedimiento de licitación.
A) Módulo de convocatorias y licitaciones
Este módulo permitirá gestionar la información relacionada con los procesos de licitación.
Debe incluir:
Registro de nuevas licitaciones
Publicación de convocatorias
Administración de bases de licitación
Registro de fechas importantes del proceso
Gestión de documentos oficiales
Publicación de aclaraciones y modificaciones
También permitirá mostrar información como:
Número de licitación
Tipo de procedimiento (licitación pública, invitación restringida, adjudicación directa)
Dependencia convocante
Objeto de la obra o servicio
Fecha de apertura de propuestas
Estado del proceso
B) Módulo de registro de proveedores
Este módulo permitirá gestionar el registro y control de empresas o proveedores interesados en participar en licitaciones.
Debe incluir:
Registro de proveedores
Captura de información empresarial
Carga de documentos legales
Validación de requisitos
Historial de participación en licitaciones
Control de estatus del proveedor
La información registrada puede incluir:
Nombre de la empresa
Representante legal
Registro fiscal
Domicilio
Especialidad en obra o servicios
Documentación legal y técnica
C) Módulo de recepción de propuestas
Este módulo permitirá a los proveedores registrados enviar sus propuestas para participar en una licitación.
Debe incluir:
Carga de propuesta técnica
Carga de propuesta económica
Documentación complementaria
Registro de fecha de entrega
Validación de requisitos obligatorios
Confirmación de recepción de propuesta
Además, el sistema deberá permitir almacenar todos los documentos relacionados con cada propuesta.
D) Módulo de evaluación de propuestas
Este módulo permitirá a los responsables del proceso evaluar las propuestas presentadas.
Debe permitir:
Comparación de propuestas
Evaluación técnica
Evaluación económica
Registro de observaciones
Generación de dictámenes
Registro de resultados de evaluación
El sistema deberá mantener un historial de evaluaciones para garantizar transparencia y control administrativo.
E) Módulo de adjudicación y contratación
Este módulo permitirá formalizar la decisión final del proceso de licitación.
Debe incluir:
Registro de proveedor ganador
Generación del acta de adjudicación
Registro del contrato
Administración de documentos del contrato
Seguimiento del proceso de contratación
También deberá permitir consultar el historial de contratos adjudicados.
F) Módulo de reportes y seguimiento
Este módulo permitirá obtener información estadística y administrativa sobre los procesos de contratación.
Debe permitir generar reportes como:
Licitaciones realizadas
Licitaciones en proceso
Licitaciones adjudicadas
Participación de proveedores
Historial de contratos
Reportes por periodo o dependencia

Resultado del proceso

5) Reglas de calidad y control del sistema
Para garantizar la correcta gestión de los procesos de licitación, el sistema deberá cumplir con los siguientes lineamientos:
✅ Validación de datos obligatorios en cada registro.
✅ Control de acceso según rol del usuario.
✅ Registro de auditoría de todas las acciones realizadas en el sistema.
✅ Control de versiones de documentos.
✅ Seguridad en el almacenamiento de información.
✅ Trazabilidad completa del proceso de licitación.
Asimismo, el sistema deberá permitir mantener un historial de cambios, indicando qué usuario realizó cada modificación y en qué momento.
6) Funciones adicionales recomendadas
Para mejorar la funcionalidad del sistema, se pueden incluir características adicionales como:
📊 Panel de indicadores de gestión de licitaciones.
🔔 Sistema de notificaciones automáticas a proveedores.
📄 Generación automática de documentos oficiales.
🔍 Búsqueda avanzada de licitaciones.
🔗 Integración con sistemas institucionales.
📤 Exportación de información en PDF o Excel.
🌐 Panel de transparencia pública.
